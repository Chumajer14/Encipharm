1. Resumen ejecutivo

Enci Ventas evoluciono desde un repositorio inicial hacia un MVP web de gestion comercial con backend FastAPI, frontend React/Vite, autenticacion Firebase, persistencia en Firestore, control de roles, CRM de clientes, flujo comercial y preparacion de despliegue frontend.

EPIC 1 cerrado Base backend, Firebase Auth, JWT, usuario autenticado y documentacion inicial.
EPIC 2 cerrado CRM clientes, usuarios/roles, Firestore, CSV, dashboards base e integracion frontend.
EPIC 3 cerrado Interacciones, oportunidades, propuestas, detalle comercial y metricas de pipeline.
EPIC 4 iniciado Hardening de sesion, pagina 404, estados vacios y estabilidad de previews.

La direccion del proyecto se mantuvo como MVP web primero. Se dejaron fuera de la fase actual: IA/RAG, Flutter nativo, ERP SAP, Google Calendar, inteligencia competitiva avanzada, reportes extendidos y migracion historica de datos.

4. Funcionalidades actuales del sistema

Autenticacion y sesion

  • Login web con Firebase Auth y Google SSO.
  • Validacion de ID Token/JWT en FastAPI mediante Firebase Admin.
  • Sincronizacion del usuario autenticado en Firestore mediante /auth/login y /auth/register.
  • Endpoint /me para retornar uid, email, rol y estado activo.
  • Manejo frontend de sesion expirada ante respuestas 401.

Usuarios y roles

  • Roles disponibles: admin, supervisor y vendedor.
  • Administracion backend de usuarios, roles y estado.
  • Permisos diferenciados para clientes, usuarios, dashboards y flujo comercial.
  • Selector temporal de rol para facilitar revision del MVP durante previews.

CRM de clientes

  • Listado, busqueda, filtros, creacion, detalle, edicion y baja logica de clientes.
  • Persistencia real en Cloud Firestore.
  • Restriccion por vendedor: cada vendedor ve y opera sus propios clientes.
  • Supervisor y admin pueden consultar el consolidado.
  • Importacion CSV backend con validacion all-or-nothing, limite de tamano y reporte por fila.
  • Normalizacion defensiva de clientes antiguos y telefonos legacy.

Flujo comercial

  • Interacciones de cliente: llamada, visita, correo y reunion.
  • Pipeline de oportunidades con etapas nuevo, contactado, cotizacion, negociacion, ganado y perdido.
  • Propuestas con estados borrador, enviada, aceptada y rechazada.
  • Calculo automatico de descuento y monto total en propuestas.
  • Relacion obligatoria entre propuesta, oportunidad y cliente.
  • Detalle de oportunidad con interacciones y propuestas vinculadas.
  • Historial comercial embebido en el detalle de cliente.

Dashboards y metricas

  • Dashboard vendedor con metricas de clientes visibles.
  • Dashboard supervisor/admin con consolidado de clientes, estados, rubros y regiones.
  • Metricas comerciales: total de oportunidades, valor de pipeline, total de propuestas y valor aceptado.
  • Conteos por etapa de oportunidad y por estado de propuesta.
  • Estados vacios explicitos para evitar paneles ambiguos.

Testing, documentacion y despliegue

  • Consola personalizada en /docs con login Google y uso del token en Swagger.
  • OpenAPI disponible en /openapi.json.
  • Documentacion viva en README y docs/api, docs/sprints, docs/qa y docs/evm.
  • Pruebas backend para permisos, CRM, seguridad y flujo comercial.
  • Lint y build frontend documentados como parte de QA.
  • Configuracion Vercel para preview del frontend React/Vite.

5. Cambios agregados, quitados y modificados

Agregado

  • Monorepo con Backend, frontend y docs.
  • Backend FastAPI con uv, Pydantic, Firebase Admin y Firestore.
  • Frontend React/Vite con rutas protegidas y pantallas comerciales.
  • Autenticacion Google SSO, JWT y autorizacion por rol.
  • CRM completo de clientes y API documentada.
  • Gestion backend de usuarios y roles.
  • Importacion CSV de clientes con validacion previa.
  • Interacciones, oportunidades, propuestas y dashboards comerciales.
  • Rate limiting, limite de tamano de request y headers de seguridad.
  • Configuracion de Vercel y documentacion de CORS para previews.

Quitado o diferido

  • node_modules versionado fue removido del repositorio.
  • Swagger UI estandar fue reemplazado por una consola personalizada en /docs.
  • Routers legacy, servicios duplicados, mocks antiguos, assets de plantilla y tests locales obsoletos fueron limpiados.
  • La migracion de datos historicos quedo fuera de EPIC 4.
  • IA/RAG, Flutter, ERP SAP, Google Calendar, inteligencia competitiva avanzada y reportes extendidos quedaron fuera del MVP web.

Modificado o endurecido

  • CORS paso a soportar origenes locales, previews remotos y expresion regular acotada.
  • Clientes legacy se normalizan antes de responder para evitar ResponseValidationError.
  • Telefonos chilenos y legacy se sanitizan para mantener compatibilidad con datos existentes.
  • Frontend muestra errores reales de API en la pantalla de clientes.
  • Se agregaron validaciones contra formula injection en campos exportables.
  • Se reforzo el manejo de token expirado y rutas inexistentes.
  • Propuestas ahora requieren oportunidad asociada y validan consistencia con cliente.

El backend concentra las reglas de autorizacion y consistencia. El frontend consume la API con Bearer token, por lo que los datos visibles dependen del rol y del usuario autenticado.

7. Estado actual y pendientes

Al corte del 11 de mayo de 2026, el sistema ya cubre los flujos principales del MVP comercial: autenticacion, CRM, roles, clientes, dashboards, interacciones, oportunidades y propuestas. La linea activa de trabajo esta enfocada en estabilizar previews, normalizar registros antiguos y cerrar hardening.

Pendientes tecnicos declarados en la documentacion: hardening final de navegacion por rol, plan QA/E2E de flujos criticos, validacion visual manual de dashboards y UAT. La migracion historica de datos no forma parte del alcance inmediato.
